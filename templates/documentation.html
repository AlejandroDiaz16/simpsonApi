Searching

All resources support Search parameters that filter the information you got. This allows you to make queries like:

http://sergobustos1.pythonanywhere.com/api/characters?status=alive

All searches will use case-sensitive partial matches on the set of search field. To see the set of search fields for each Resource, check out the individual documentation.

Characters

A character resource is an individual person or character within the Simpsons Universe.


Endpoints:

  • /characters

    -- get all the character in the resource

  • /characters/id

    -- get a specific character

Example request:

Example response:

Attributes:

  • name string

    -- The name of this character.

  • age string

    -- The age of this character.

  • status string

    -- The status of this character. Will be "alive" or "dead".

  • gender string

    -- The gender of this character. Will be "Male" or "Female".

  • image string

    -- The link od our server where is located the .png image for this character

  • occupation string

    -- The last known occupation of this character.

  • voice string

    -- The name of the person who makes the character's voice.

  • created string

    -- the date when was created this character. Will be "mm/dd/yyyy"

  • edited string

    -- the date when was edited this character. Will be "mm/dd/yyyy"

Search Parameters

  • name

    -- filter by name

  • status

    -- filter by status

  • apparition

    -- Will show you all the characters that appears for first time into an eppisode.

  • age

    -- filter by age. Will show you all the characters that are older than the age parameter.

  • -age

    -- filter by age. Will show you all the characters that are younger than the age parameter.

  • gender

    -- filter by gender

Locations

A location resource is a building with a specific activity like banks, stores, public building.


Endpoints:

  • /locations

    -- get all the locations

  • /location/id

    -- get a specific location

Example request:

Example response:

Attributes:

  • name string

    -- The name of this location.

  • type string

    -- The type of the location. Example: goverment building, banks or market.

  • Use string

    -- The specific use. Example: Bank, CityHall, Bar, Store.

  • created string

    -- the date when was created this location. Will be "mm/dd/yyyy"

  • edited string

    -- the date when was edited this location. Will be "mm/dd/yyyy"

Search Parameters

  • name

    -- filter by name

  • type

    -- filter by type

  • use

    -- filter by use

Cities

A city resource is a specific place in the simpsons universe.


Endpoints:

  • /cities

    -- get all the cities

  • /cities/id

    -- get a specific city

Example request:

Example response:

Attributes:

  • name string

    -- The name of this city.

  • population string

    -- The estimated population of this city.

  • image string

    -- The link where is located the image of this city.

  • created string

    -- the date when was created this city. Will be "mm/dd/yyyy"

  • edited string

    -- the date when was edited this city. Will be "mm/dd/yyyy"

Search Parameters

  • name

    -- filter by name

  • -population

    -- filter the cities that has population under the parameter.

  • population

    -- filter the cities that has more population than the parameter.

An episode resource is a chapter of the simpsons universe

    -- get all the episodes

  • /episodes/id

    -- get a specific episode

Example request:

Example response:

Attributes:
